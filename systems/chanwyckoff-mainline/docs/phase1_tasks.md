# 第一阶段实施任务清单

## 阶段目标

第一阶段目标是跑通完整研究闭环：

```text
数据入库 -> 市场环境判断 -> 题材强度 -> 趋势容量核心 -> 30 分钟三买扫描 -> 回测 -> 今日作战台展示
```

暂不接真实交易下单。半自动交易只预留数据模型和人工确认状态。

## 里程碑 1：工程骨架

交付物：

- FastAPI 后端工程。
- Next.js + React + TypeScript 前端工程。
- MySQL 连接配置。
- SQLAlchemy/Alembic 初始化。
- APScheduler 初始化。
- pytest 测试框架。
- 本地开发启动说明。

关键任务：

- 初始化后端目录结构。
- 初始化前端目录结构。
- 建立统一配置模块。
- 建立数据库连接和迁移流程。
- 提供 `/health` API。
- 提供前后端本地启动脚本。

完成标准：

- 后端能启动并访问 `/health`。
- 前端能启动并调用后端健康检查。
- Alembic 能创建第一版表结构。

## 里程碑 2：数据模型与入库

交付物：

- 股票基础信息表。
- 交易日历表。
- 日线行情表。
- 30 分钟行情表。
- 指数行情表。
- 概念/题材表。
- 概念成分表。
- 板块行情快照表。
- 本地通达信快照表。

关键任务：

- 设计 MySQL 表结构。
- 实现 TickFlowProvider 接口。
- 实现 AkshareThemeProvider 接口。
- 实现 TdxSnapshotProvider 轻量导入。
- 支持按日期范围批量拉取日线和 30 分钟 K。
- 支持前复权数据入库。
- 支持概念板块、成分和板块行情入库。
- 支持数据拉取任务的幂等更新。

完成标准：

- 能拉取指定股票的日线和 30 分钟前复权 K 线。
- 能拉取指定日期的概念板块和成分。
- 能导入本地通达信快照，但核心流程不依赖它。
- 数据重复导入不会产生重复记录。

## 里程碑 3：股票池与市场环境

交付物：

- 主板基础股票池。
- 流动性过滤器。
- 市场环境评分。
- `risk_on / neutral / risk_off` 状态。

关键任务：

- 排除 ST、停牌、退市风险、创业板、科创板、北交所。
- 实现成交额、市值、换手过滤。
- 实现单边下跌过滤。
- 接入上证指数和全 A 指数/全市场赚钱效应。
- 计算上涨家数、跌停数、成交额变化、强势板块数量。
- 输出统一 `market_regime`。

完成标准：

- 给定交易日，系统能输出可交易主板股票池。
- 给定交易日，系统能输出市场环境分档和证据字段。
- `risk_off` 状态下普通新信号会被压制或降级。

## 里程碑 4：题材主线与趋势容量核心

交付物：

- 概念/题材强度评分。
- 穿越主线识别。
- 趋势容量核心排名。
- 主归因 + 多标签归因。

关键任务：

- 计算板块近 3/5/10/20 日相对强度。
- 计算板块成交额相对 20 日均值放大。
- 计算板块上涨家数、涨停数、创新高数。
- 标注 `resistant_theme`、`emerging_leader`、`confirmed_mainline`、`exhausted_theme`。
- 为个股选择最高强度概念作为主归因。
- 按 30/25/20/15/10 权重计算趋势容量核心。

完成标准：

- 今日作战台能展示强势题材和穿越主线候选。
- 每个题材能展示趋势容量核心票排名。
- 后排低容量票不会因单日涨幅被错误排到最前。

## 里程碑 5：30 分钟结构识别

交付物：

- 30 分钟分型识别。
- 工程化笔生成。
- 工程化中枢识别。
- 统计平台兜底识别。
- 平台/中枢质量评分。

关键任务：

- 实现 K 线包含关系和分型去噪规则。
- 实现工程化笔生成。
- 识别至少 3 笔重叠形成的候选中枢。
- 计算中枢上沿、下沿、中轴、持续时间、振幅、重叠率、上沿测试次数。
- 实现 N 根 K 的统计平台识别。
- 输出结构标签和质量分。

完成标准：

- 给定一段 30 分钟 K 线，能稳定输出平台/中枢边界。
- 对无结构的单边下跌样本不会误报大量平台。
- 单元测试覆盖典型平台、收敛平台、假平台、单边走势。

## 里程碑 6：三买状态机与威科夫评分

交付物：

- `proto_3buy`、`confirmed_3buy`、`failed_3buy` 状态机。
- 突破确认规则。
- 回踩确认规则。
- 威科夫背景/特征/预判评分。

关键任务：

- 实现收盘突破 + 放量 + K 线质量判断。
- 实现突破后 1-8 根 30 分钟 K 的回踩跟踪。
- 实现容忍带判断，支持 0.5ATR 或 1%-2%。
- 实现回踩缩量和承接质量评分。
- 实现失败三买识别。
- 实现威科夫背景、特征、预判三段式证据字段。
- 将 `wyckoff_score` 映射为信号等级。

完成标准：

- 突破优秀但未回踩确认时输出 `proto_3buy`。
- 回踩不有效回中枢且缩量承接时输出 `confirmed_3buy`。
- 放量跌回中枢或超时未站稳时输出 `failed_3buy`。
- 每个信号都有结构证据和量价证据。

## 里程碑 7：回测引擎

交付物：

- 结构策略回测。
- 题材增强回测。
- 交易成本、滑点、涨跌停、停牌、容量约束。
- 小范围网格搜索。
- 稳健性报告。

关键任务：

- 支持 2019 至今结构策略回测。
- 支持题材历史可靠区间的增强回测。
- 标注使用当前概念成分回填历史的近似回测。
- 用下一根 30 分钟 K 的合理成交价模拟成交。
- 计入佣金、规费、印花税和滑点。
- 处理涨停不能买、跌停不能卖、停牌不可成交。
- 实现单票、单板块、总仓位限制。
- 输出胜率、均值/中位收益、盈亏比、最大回撤、持仓周期。
- 输出 `proto_3buy` 和 `confirmed_3buy` 分层表现。
- 输出 `wyckoff_score` 分段表现。
- 输出参数稳健性报告。

完成标准：

- 能跑指定日期范围和参数组合。
- 能证明高分信号是否优于低分信号。
- 能发现过拟合参数和样本集中风险。
- 回测报告明确标注题材数据可靠性。

## 里程碑 8：今日作战台

交付物：

- 今日作战台页面。
- 市场环境卡片。
- 穿越主线列表。
- 趋势容量核心列表。
- 三买候选列表。
- 信号证据详情。

关键任务：

- 实现后端 API：市场环境、题材主线、核心票、三买信号、信号详情。
- 前端展示 `risk_on / neutral / risk_off`。
- 前端展示板块强度、成交额放大、赚钱效应。
- 前端展示核心票排名和主归因概念。
- 前端展示 `proto_3buy`、`confirmed_3buy`、`failed_3buy`。
- 前端展示建议动作：轻仓、加仓、观察、过滤。
- 支持按题材、信号等级、评分、成交额筛选。

完成标准：

- 打开今日作战台即可回答今天能不能进攻。
- 能看到当前主线候选和对应核心票。
- 能看到每个信号的结构、量价和风控证据。

## 里程碑 9：复盘与 LLM 辅助

交付物：

- 信号复盘记录。
- 人工确认状态。
- LLM 解释与失败样本总结。

关键任务：

- 支持记录准备买、已买、放弃、卖出。
- 支持记录人工备注。
- 支持记录信号后表现。
- LLM 异步生成背景、特征、预判摘要。
- LLM 归纳失败样本类型。
- 严格禁止 LLM 直接决定信号和仓位。

完成标准：

- 每个信号可追踪从出现到结束的全过程。
- 复盘页能看到失败原因分布。
- LLM 输出只作为解释，不改变规则结果。

## 第一阶段暂不做

- 全自动下单。
- Level2、逐笔、盘口依赖。
- 复杂机器学习评分。
- 严格正统缠论递归实现。
- 同花顺/iFinD 作为硬依赖。
- 使用未来概念成分做正式历史结论。

## 风险点

- 东方财富/AkShare 题材历史不完整，必须区分结构回测和题材增强回测。
- 30 分钟历史数据量大，TickFlow 拉取需要分页和缓存。
- 概念成分变更若不能历史回溯，会影响 2019 至今题材统计。
- 工程化中枢规则需要大量样本校验，避免误报。
- 偏进攻仓位容易放大假突破损失，必须保留市场环境闸门。
- 回测若不严格处理涨跌停、停牌和滑点，会明显高估收益。

## 推荐开发顺序

1. 工程骨架。
2. MySQL 表结构和数据入库。
3. TickFlow 日线/30 分钟接入。
4. AkShare 概念板块接入。
5. 主板股票池和市场环境。
6. 板块强度和趋势容量核心。
7. 30 分钟平台/中枢识别。
8. 三买状态机和威科夫评分。
9. 结构策略回测。
10. 今日作战台。
11. 题材增强回测。
12. 复盘和 LLM 解释。
